# Checklist – Testes de API

## Validação básica
- [ ] Status code correto (200, 201, 400, 404, 500)
- [ ] Método HTTP correto (GET, POST, PUT, DELETE)
- [ ] Headers corretos

## Regras de negócio
- [ ] Campos obrigatórios validados
- [ ] Dados inválidos retornam erro
- [ ] Mensagens claras

## Banco de dados
- [ ] Dados persistem corretamente
- [ ] Não duplica registros
- [ ] Integridade dos dados

## Segurança
- [ ] Autenticação necessária
- [ ] Não expõe dados sensíveis
